# Brand assets — Undaunted Treasure Trove

This folder contains the live brand assets used by the storefront, PWA and
admin. Almost all icon/social files are **generated** from a single master
logo, so re-branding is a one-command operation.

## Generate or regenerate everything

```bash
# Uses public/logo.png as the master source
npm run generate:brand

# Or point it at a different source file (e.g. a fresh logo handoff)
LOGO_SRC=/abs/path/to/new-logo.png npm run generate:brand
```

The script (`scripts/generate-brand-assets.mjs`) writes:

| File | Size | Purpose |
|------|------|---------|
| `logo.png` | 735×520 (master) | Header / Footer / PWA splash / admin / About page / JSON-LD logo |
| `favicon.ico` | 16+32+48 multi | Browser tab icon (legacy + modern) |
| `favicon-16x16.png` / `favicon-32x32.png` / `favicon-48x48.png` | png | Browser tab |
| `apple-touch-icon.png` | 180×180 | iOS home screen |
| `icon-{16,32,48,64,96,128,192,256,384,512}.png` | png | PWA + general purpose |
| `icon-maskable-192.png` / `icon-maskable-512.png` | png | Android adaptive icon (safe-area padded) |
| `og-image.png` | 1200×630 | Open Graph / Twitter / WhatsApp preview |
| `og-image-square.png` | 1200×1200 | Square preview (Pinterest, some IM previews) |
| `twitter-image.png` | 1200×630 | Mirrors og-image (in case you ever divergence them) |

The icon backgrounds use **`#172554` (Tailwind blue-950)** so the gold logo
reads cleanly on both light and dark device themes. Maskable variants get
an extra 38% safe-area padding so Android's adaptive-icon mask never
crops the monogram.

## Other files in this folder

- `manifest.json` — PWA manifest with full icon set + maskable variants, app
  shortcuts, share target and screenshots. Don't edit by hand for icons —
  re-run `npm run generate:brand` and it stays in sync.
- `browserconfig.xml` — Windows tile config (used when the site is pinned
  to a Windows Start screen).
- `robots.txt` — Static robots file (also generated dynamically by
  `app/robots.ts`; both should agree).
- `service-worker.js` — Custom service worker for offline support.
- `placeholder.svg` — Generic product image fallback for missing imagery.

## Hero / page imagery

The homepage and section hero backgrounds (`hero-slide-*.png`,
`page-hero-*.png`) are **not** generated by the script — drop your own
photography into this folder using those filenames. They render at
1920×1080 (homepage hero) and 1920×600 (page heroes) and are referenced
directly from the page components.
